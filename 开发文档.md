### 架构说明

```
请求 → FastAPI 路由层 (app/api/) 
            ↓
      业务逻辑层 (app/services/) 
            ↓
      数据访问层 (app/core/database.py)
       ├── SQLite (SQLAlchemy ORM) — 用户、课程、进度等关系型数据
       └── Neo4j (Cypher) — 知识图谱节点和关系
```

**关键设计决策：**

1. **双数据库存储** — 知识点同时存储在 SQLite（关系型，用于进度关联）和 Neo4j（图，用于拓扑排序和可视化）。SQLite 中的 `knowledge_points` 表通过 `neo4j_node_id` 字段与 Neo4j 节点关联。
2. **DeepSeek API 调用** — 使用 OpenAI 兼容 SDK 调用 DeepSeek，知识提取 prompt 已调优为中文教育场景。
3. **拓扑排序推荐** — 基于 Kahn 算法，从 Neo4j 的 `PREREQUISITE` 关系构建 DAG，结合用户进度生成个性化推荐。
4. **JWT 认证** — 所有需要用户身份的端点通过 `get_current_user` 依赖注入获取当前用户。

## 二、数据库表结构 (SQLite)

### users 用户表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | Integer PK | 自增主键 |
| username | String(50) UNIQUE | 用户名 |
| email | String(100) UNIQUE | 邮箱 |
| hashed_password | String(255) | bcrypt 哈希密码 |
| role | Enum(student/teacher) | 角色 |
| created_at | DateTime | 创建时间 |

### courses 课程表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | Integer PK | 自增主键 |
| title | String(200) | 课程标题 |
| description | Text | 课程描述 |
| teacher_id | Integer FK→users.id | 创建教师 |
| created_at / updated_at | DateTime | 时间戳 |

### knowledge_points 知识点表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | Integer PK | 自增主键 |
| neo4j_node_id | String(100) UNIQUE | 对应 Neo4j 节点 ID，格式 `kp_{course_id}_{order_index}` |
| course_id | Integer FK→courses.id | 所属课程 |
| name | String(200) | 知识点名称 |
| description | Text | 知识点描述 |
| order_index | Integer | 排序序号 |

### knowledge_relations 知识点关系表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | Integer PK | 自增主键 |
| course_id | Integer FK→courses.id | 所属课程 |
| source_kp_id | Integer FK→knowledge_points.id | 源知识点 |
| target_kp_id | Integer FK→knowledge_points.id | 目标知识点 |
| relation_type | Enum(prerequisite/related_to/part_of) | 关系类型 |

### documents 文档表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | Integer PK | 自增主键 |
| course_id | Integer FK→courses.id | 所属课程 |
| filename | String(255) | 原始文件名 |
| file_path | String(500) | 服务器存储路径 |
| parsed_content | Text | 解析后的纯文本 |
| status | Enum(uploaded/parsed/extracted/failed) | 处理状态 |

### user_knowledge_progress 用户进度表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | Integer PK | 自增主键 |
| user_id | Integer FK→users.id | 用户 |
| knowledge_point_id | Integer FK→knowledge_points.id | 知识点 |
| status | Enum(not_started/in_progress/mastered) | 学习状态 |
| mastery_level | Float(0.0~1.0) | 掌握程度 |
| updated_at | DateTime | 更新时间 |

### Neo4j 图模型

```
(:KnowledgePoint {
    neo4j_id: "kp_1_0",        -- 唯一标识
    name: "Python变量",         -- 知识点名称
    description: "...",         -- 描述
    course_id: 1,               -- 所属课程
    order_index: 0              -- 排序
})

关系类型：
  (A)-[:PREREQUISITE]->(B)    A 是 B 的先修知识点（学习顺序依赖）
  (A)-[:RELATED_TO]->(B)      A 与 B 内容相关
  (A)-[:PART_OF]->(B)         A 是 B 的组成部分
```

## 三、API 接口文档

> 所有接口前缀：`http://localhost:8000`
> 启动后访问 `/docs` 获取 Swagger 交互式文档

### 3.1 认证接口 (`/api/auth`)

| 方法 | 路径 | 权限 | 请求体 | 说明 |
|---|---|---|---|---|
| POST | `/api/auth/register` | 公开 | `{username, email, password, role?}` | 注册，返回 JWT |
| POST | `/api/auth/login` | 公开 | `{username, password}` | 登录，返回 JWT |
| GET | `/api/auth/me` | 登录 | — | 获取当前用户信息 |

**认证方式：** Header 中携带 `Authorization: Bearer <token>`

### 3.2 课程与图谱接口 (`/api/courses`)

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| POST | `/api/courses/` | 教师 | 创建课程 |
| GET | `/api/courses/` | 登录 | 课程列表（教师看自己的，学生看全部） |
| GET | `/api/courses/{id}` | 登录 | 课程详情（含知识点/文档数量统计） |
| PUT | `/api/courses/{id}` | 创建者 | 编辑课程 |
| DELETE | `/api/courses/{id}` | 创建者 | 删除课程 + 级联清除图谱和文件 |
| POST | `/api/courses/{id}/upload` | 创建者 | 上传文档（multipart/form-data, field: `file`） |
| POST | `/api/courses/{id}/extract` | 创建者 | 触发 DeepSeek 知识提取 |
| GET | `/api/courses/{id}/graph` | 登录 | 获取图谱（返回 `{nodes, edges}`，适配 AntV G6） |
| DELETE | `/api/courses/{id}/graph` | 创建者 | 清除图谱 |

**知识提取流程：**
```
上传文档 → parser.py 解析文本 → 存入 documents 表
触发提取 → extractor.py 调用 DeepSeek → 返回知识点 + 关系
         → graph_ops.py 写入 Neo4j
         → 同步写入 SQLite (knowledge_points + knowledge_relations)
```

### 3.3 学习进度接口 (`/api/learning`) — 功能3

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/api/learning/progress/{course_id}` | 登录 | 获取用户在该课程每个知识点的状态 |
| POST | `/api/learning/progress` | 登录 | 更新单个知识点状态 `{knowledge_point_id, status, mastery_level}` |
| POST | `/api/learning/progress/batch` | 登录 | 批量更新 `[{knowledge_point_id, status, mastery_level}, ...]` |
| GET | `/api/learning/stats/{course_id}` | 登录 | 进度统计（总数、已掌握、学习中、未开始、百分比） |

### 3.4 个性化推荐接口 (`/api/learning`) — 功能4

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/api/learning/recommend/{course_id}` | 登录 | 完整推荐结果（拓扑排序后的全部路径 + 可学列表 + 推荐前5） |
| GET | `/api/learning/next/{course_id}?count=3` | 登录 | 简版：返回下一个推荐学习的 N 个知识点 |

**推荐算法说明：** 见 `backend/app/services/recommender.py`，核心逻辑：
1. 从 Neo4j 提取 `PREREQUISITE` 关系构建邻接表
2. Kahn 算法拓扑排序
3. 结合用户在 `user_knowledge_progress` 表中的掌握状态
4. 筛选所有先修条件已满足的"可学"知识点

## 五、开发环境搭建指南

### 前置条件

- Python 3.10+
- Node.js 18+
- Neo4j AuraDB 免费实例（[注册地址](https://neo4j.com/cloud/aura/)）
- DeepSeek API Key（[申请地址](https://platform.deepseek.com/)）

### 后端启动步骤

```bash
# 1. 进入后端目录
cd backend

# 2. 创建虚拟环境
python -m venv venv
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置环境变量
cp ../.env.example .env
# 编辑 .env，填入你的 DeepSeek API Key 和 Neo4j 连接信息

# 5. 启动服务
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 前端启动步骤

```bash
cd frontend
npm install
npm run dev
# 默认访问 http://localhost:5173
```

### 验证后端是否正常

```bash
# 健康检查
curl http://localhost:8000/

# 查看 API 文档
open http://localhost:8000/docs
```

## 八、常见问题

**Q: 启动后端时提示 `No module named 'xxx'`？**
A: 确保已激活虚拟环境并执行 `pip install -r requirements.txt`。

**Q: 知识提取失败？**
A: 检查 `.env` 中 `DEEPSEEK_API_KEY` 是否正确，以及账户余额是否充足。

**Q: Neo4j 连接失败？**
A: 检查 AuraDB 实例是否在运行，以及 `NEO4J_URI` 格式是否正确（末尾不需要端口号，AuraDB 使用 `neo4j+s://` 协议）。

**Q: 上传文档后提取不到知识点？**
A: 检查文档解析状态（`documents.status`），若为 `failed` 则文档格式可能有问题。PDF 需要是文本型而非扫描型。
